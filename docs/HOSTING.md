# SENVIROX — Hosting Guide (100% deployable stack)

This guide deploys **Modules 1–2** (Foundation + Auth & RBAC) as a production Docker stack:

- `nginx` TLS edge (:80/:443)
- `frontend` Next.js console
- `backend` Express API
- `redis` auth sessions / cache
- durable users via **file volume** (default) or **Supabase**

---

## What you get

| Path | Purpose |
|------|---------|
| `https://YOUR_DOMAIN/` | Console (login/register/account/health) |
| `https://YOUR_DOMAIN/api/v1/*` | API |
| `https://YOUR_DOMAIN/health` | Health JSON |
| Internal `/metrics` | Prometheus (token required; blocked at nginx edge) |

---

## Server requirements

- Linux VPS (2 vCPU / 2 GB RAM minimum)
- Docker Engine 24+ and Docker Compose v2
- Domain DNS **A/AAAA** → server IP
- Open ports **80** and **443**

---

## 15-minute deploy

```bash
# 1) Copy project to server
cd /opt
unzip SENVIROX_Hostable_Product.zip
cd senvirox

# 2) Generate production secrets + .env
node scripts/generate-secrets.mjs app.yourdomain.com

# 3) TLS certificates
# Option A — temporary self-signed (works immediately; browsers warn)
sh scripts/gen-dev-certs.sh app.yourdomain.com

# Option B — real certs (recommended)
# Place PEMs at:
#   deploy/ssl/fullchain.pem
#   deploy/ssl/privkey.pem
# (from Let's Encrypt / Cloudflare / ACM)

# 4) Launch
chmod +x scripts/deploy.sh scripts/gen-dev-certs.sh
./scripts/deploy.sh
```

Open `https://app.yourdomain.com/register` — **first user becomes `super_admin`**.

## Environment modes

| `USER_STORE` | Behavior |
|--------------|----------|
| `file` (default) | Users in Docker volume `backend-data` → `/data/users.json` |
| `supabase` | Postgres table `app_users` (run `database/001_app_users.sql`) |
| `auto` | Supabase if credentials present, else file |
| `memory` | Tests only — blocked in production |

---

## Required production secrets

Generated by `scripts/generate-secrets.mjs`:

- `JWT_SECRET`
- `REDIS_PASSWORD`
- `METRICS_TOKEN`
- `CORS_ORIGIN` / `NEXT_PUBLIC_APP_URL` / `NEXT_PUBLIC_API_URL` (HTTPS domain)

API **refuses to boot** in production if these are weak/missing.

## Operations

```bash
# Status / logs
docker compose ps
docker compose logs -f backend

# Update
git pull   # or replace release zip
docker compose build
docker compose up -d

# Backup users (file store)
docker compose exec backend cat /data/users.json > backup-users.json

# Backup redis
docker compose exec redis redis-cli -a "$REDIS_PASSWORD" BGSAVE
```

## Security baseline included

- TLS termination + HSTS
- Security headers
- Auth rate limits (app + nginx)
- Non-root containers, `no-new-privileges`
- Redis password + no host publish
- Metrics token; `/metrics` not exposed on edge
- scrypt passwords + rotating refresh tokens

## Optional: Supabase users

1. Create Supabase project  
2. Run `database/001_app_users.sql`  
3. Set in `.env`:

```bash
USER_STORE=supabase
SUPABASE_URL=https://xxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=...
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
```

4. `docker compose up -d backend`
